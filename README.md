# تطبيق مترو القاهرة — نسخة جاهزة للنشر (v2.0)

تطبيق ويب تقدمي (PWA) كامل لتخطيط رحلات مترو القاهرة، مونوريل شرق النيل، والقطار الكهربائي الخفيف — يعمل بدون إنترنت بعد أول زيارة.

---

## 📦 الملفات

| الملف | الوظيفة |
|---|---|
| `index.html` | التطبيق نفسه (الواجهة + قاعدة البيانات + الخوارزميات) |
| `manifest.json` | بيانات التثبيت على الشاشة الرئيسية (PWA) |
| `sw.js` | Service Worker — يخزن التطبيق في ذاكرة المتصفح ليعمل تحت الأرض بدون نت |
| `icon-192.png`, `icon-512.png`, `icon-512-maskable.png` | أيقونات التطبيق |
| `apple-touch-icon.png`, `favicon-32.png` | أيقونات آيفون والمتصفح |

ضع كل هذه الملفات في **نفس المجلد** على السيرفر — لا تغيّر مساراتها، لأن `index.html` و`manifest.json` و`sw.js` يشيرون إلى بعضها بمسارات نسبية (`./`).

---

## ✅ ما تم تنفيذه من قائمة "الخطوات القادمة"

**1. اكتمال قاعدة البيانات (محطات حقيقية ودقيقة)**
تم استبدال الأسماء التقريبية (مثل R1 / R2 / R3) ببيانات رسمية حقيقية مجمّعة من تصريحات وزارة النقل ومصادر إخبارية حتى يونيو 2026:
- الخطوط 1 و2 و3 (بكل فروعها الثلاثة) — كانت مكتملة بالفعل وتم التحقق منها.
- **مونوريل شرق النيل**: 22 محطة كاملة بالاسم الرسمي (بدلاً من R1/R2/R3 الوهمية)، مع تمييز المحطات التي لا تزال قيد التشغيل التجريبي.
- **القطار الكهربائي الخفيف (LRT)**: تمت إضافة فرعَي الخط الكاملين (الفرع الجنوبي إلى العاصمة، والفرع الشمالي إلى العاشر من رمضان) بدلاً من مسار واحد مبسّط — إجمالي 19 محطة رسمية.
- إجمالي الشبكة الآن **122 محطة فعلية متصلة فعليًا بالخريطة**.
- تم اكتشاف وحل تعارض حقيقي في الأسماء: هناك محطتان مختلفتان تمامًا (إحداهما بالمترو والأخرى بالمونوريل) تحملان اسم "هشام بركات" في منطقتين مختلفتين من القاهرة — تم تمييزهما حتى لا يخلط التطبيق بينهما كمحطة تبادلية وهمية.

**2. تفعيل العمل بدون إنترنت (Service Worker حقيقي)**
ملف `sw.js` يقوم فعليًا بما كان مخططًا له: تخزين الصفحة وكل الأصول في Cache Storage، مع تحديث تلقائي بالخلفية، وعرض النسخة المخزنة فور انقطاع الشبكة. تم أيضًا إضافة:
- `manifest.json` فعلي مع أيقونات (لم يكن موجودًا، كان فقط Meta Tags لآيفون).
- زر تثبيت تلقائي على أندرويد/ديسكتوب (`beforeinstallprompt`)، وتنبيه يدوي لمستخدمي آيفون (لأن Safari لا يدعم هذا الحدث).
- مؤشر "متصل / غير متصل" في الشريط العلوي.

**3. حساب الوقت المتقدم**
- صار وقت الرحلة يُحسب بمتوسط دقائق مختلف لكل نوع خط (مترو 2.2 دقيقة/محطة، مونوريل 3.2، قطار خفيف 5.5 لطول المسافات بين محطاته).
- تمت إضافة **5 دقائق افتراضية** عند كل تحويل بين الخطوط كما طُلب، مع **حالات خاصة لمحطات معروفة بطول مسافة المشي** فيها أكثر من ذلك (العتبة 8 دقائق، السادات 7 دقائق، جامعة القاهرة وعدلي منصور ومدينة الفنون والثقافة 6 دقائق).

## 🎨 تحسينات إضافية على التصميم والتجربة

- **بحث ذكي عن المحطات**: قائمة منبثقة تفاعلية تعرض نقاط ألوان الخطوط لكل محطة، مع دعم لوحة المفاتيح.
- **شريط ملخص الرحلة**: عرض المسار بطابع لوحات العرض الرقمية بالمحطات (خط → عدد محطات → تحويل → خط...).
- **تذاكر متعددة بدقة**: المترو (الخطوط 1/2/3) تذكرة واحدة موحدة، بينما المونوريل والقطار الخفيف نظام تذاكر مستقل — يعرض التطبيق الآن تكلفة كل نظام تذاكر بشكل منفصل وواقعي إذا كانت الرحلة تجمع بين أكثر من خط.
- **تنبيه شفافية**: أي مسار يتضمن محطات لم تُفتح بعد رسميًا (مثل امتداد مونوريل شرق النيل أو فروع القطار الخفيف البعيدة) يظهر له تنبيه واضح بذلك.
- **الرحلات الأخيرة**: حفظ آخر 5 رحلات على جهاز المستخدم نفسه لإعادة البحث بضغطة واحدة.
- **إصلاح ترتيب القائمة السفلية على الموبايل**: كانت أيقونات التنقل تظهر فعليًا في الأعلى بسبب ترتيب العناصر في الكود رغم أن تصميمها (الحدود والمساحة الآمنة) مخصص لشريط سفلي — تم تصحيح هذا.
- تباين ألوان أفضل لتسميات الخطوط الصفراء/الفاتحة (نص أغمق فوق الأصفر بدل الأبيض دائمًا).
- دعم `prefers-reduced-motion` وحالات `focus` واضحة لإمكانية الوصول.
- دليل الشبكة أصبح يضم كل الخطوط الخمسة بحالتها التشغيلية، إضافة لذكر مونوريل غرب النيل (٦ أكتوبر) والخط الرابع كخطوط "قيد الإنشاء" بشكل شفاف دون اختراع بيانات محطات غير مؤكدة لها رسميًا.

## 🚀 النشر (5 دقائق)

التطبيق ملفات استاتيكية بالكامل (لا يحتاج سيرفر/قاعدة بيانات)، لكنه **يجب أن يُفتح عبر HTTPS** (أو `localhost` للتجربة) ليعمل تثبيت PWA والعمل بدون إنترنت — فتحه مباشرة بصيغة `file://` لن يُفعّل الـ Service Worker.

اختر أي من الخيارات التالية:

### الخيار 1: Netlify (بالسحب والإفراج — أسهل خيار)
1. اذهب إلى [app.netlify.com/drop](https://app.netlify.com/drop)
2. اسحب مجلد الملفات كله وأفلته في الصفحة.
3. ستحصل فورًا على رابط HTTPS جاهز.

### الخيار 2: GitHub Pages
```bash
# داخل مجلد المشروع
git init
git add .
git commit -m "نشر تطبيق مترو القاهرة"
git branch -M main
git remote add origin <رابط-المستودع-الخاص-بك>
git push -u origin main
```
ثم من إعدادات المستودع على GitHub: **Settings → Pages → Source: main branch** واحفظ. سيصبح الرابط: `https://<اسم-المستخدم>.github.io/<اسم-المستودع>/`

### الخيار 3: Firebase Hosting
```bash
npm install -g firebase-tools
firebase login
firebase init hosting   # اختر مجلد الملفات الحالي كـ public directory
firebase deploy
```

## 🔄 عند نشر تحديث مستقبلي

غيّر قيمة `CACHE_VERSION` في أول سطور `sw.js` (مثلاً من `metro-cairo-v1` إلى `metro-cairo-v2`) قبل كل نشر جديد، حتى يحذف Service Worker النسخة القديمة المخزنة في أجهزة المستخدمين ويحمّل التحديث.

## ⚠️ ملاحظات مهمة حول دقة البيانات

- جرى تجميع كل بيانات المحطات والتعريفات من تصريحات وزارة النقل والهيئة القومية لمترو الأنفاق ومصادر إخبارية رسمية، **حتى 22 يونيو 2026**.
- **تعريفة القطار الكهربائي الخفيف (LRT)** غير معروضة بالأرقام لعدم توافر تسعيرة موحّدة مؤكدة لكل محطاته وقت كتابة هذا التحديث؛ التطبيق يعرض "حسب المسافة" بدلاً من رقم قد يكون غير دقيق.
- مونوريل غرب النيل (6 أكتوبر) والخط الرابع للمترو **لم يُدرجا في خوارزمية التخطيط** لأنهما لا يزالان قيد الإنشاء ولا توجد قائمة محطات نهائية مؤكدة رسميًا لهما بعد — تم ذكرهما فقط في صفحة "الدليل" بشكل شفاف. عند توفر بيانات رسمية كاملة، يمكن إضافتهما بسهولة إلى متغير `PATHS` داخل كود `index.html` بنفس الطريقة المستخدمة للخطوط الحالية.
- يُنصح بمراجعة الأسعار ومواعيد القطارات الدقيقة من الشاشات والقنوات الرسمية قبل السفر، لأنها قابلة للتغيير دون إشعار مسبق.
