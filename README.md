تطبيق ويب تقدمي (PWA) متكامل مصمم خصيصاً لتسهيل حركة الملاحة اليومية للركاب في القاهرة الكبرى. يربط التطبيق بين **خطوط المترو الثلاثة**، و**مونوريل شرق النيل**، و**القطار الكهربائي الخفيف (LRT)** في واجهة تفاعلية موحدة فائقة السرعة تعمل بالكامل **دون الحاجة للاتصال بالإنترنت**.

---

## 🌟 الميزات الأساسية | Key Features

*   **خوارزمية دايجسترا الملاحية الذكية (Dijkstra's Route Optimization):** محرك ملاحة متطور يبحث عن المسار الأقصر مع تطبيق عقوبة افتراضية (Transfer Penalty) عند تبديل الخطوط، لضمان اقتراح المسارات الأكثر راحة وعملية للمستخدم وتفادي التحويلات المنهكة.
*   **تسعير فوري متكامل لعام 2026 (Live Ticket Calculator):** حساب تلقائي لقيمة التذاكر بناءً على شرائح الأسعار الرسمية المحدثة في مارس 2026 لكافة شبكات النقل (مترو، مونوريل، قطار خفيف) بشكل فوري ودقيق.
*   **معالجة مرنة للنصوص العربية (Bilingual Arabic Search & Normalization):** نظام داخلي ذكي لتوحيد الحروف العربية وإزالة علامات التشكيل، مما يجعل واجهة البحث متسامحة مع الأخطاء الإملائية الشائعة لأسماء المحطات (مثل: أ، إ، آ، هـ، ة، ي، ى).
*   **تصميم زجاجي عصري (Premium Glassmorphic UI):** واجهة مستخدم متجاوبة تماماً (Responsive) تحاكي أفضل التطبيقات المصرفية الرقمية العالمية، وتدعم شريط التنقل السفلي للهواتف والشريط الجانبي للشاشات الكبيرة.
*   **جاهز للعمل دون إنترنت (Offline-First Ready):** معزز بملفات الخدمة (Service Workers) وتقنيات التخزين المؤقت للبيانات (Cache Storage API) ليتمكن المستخدم من تخطيط رحلته بالكامل أثناء تواجده تحت الأرض داخل أنفاق المترو.
*   **حفظ الرحلات الأخيرة (Local Storage Cache):** يسجل التطبيق تلقائياً آخر 5 رحلات بحث عنها المستخدم لتسهيل الوصول إليها بضغطة زر واحدة لاحقاً.

## 🛠️ البنية التقنية ومكونات النظام | Tech Stack

*   **HTML5:** لضمان بنية دلالية (Semantic Layout) مهيأة لمحركات البحث وإمكانية الوصول (Accessibility - WCAG).
*   **CSS3 Custom Properties:** استخدام المتغيرات الديناميكية لإدارة الألوان والحركات البصرية (Smooth Neon Glowing Backgrounds).
*   **Vanilla ES6+ JavaScript:** المنطق البرمجي الأساسي للملاحة وبناء الرسم البياني (Graph Representation) دون الاعتماد على أي مكتبات خارجية ثقيلة لضمان كفاءة التحميل.
*   **Service Workers:** لتوفير استجابة فورية للتطبيق وعمليات التخزين المؤقت للمستندات والخطوط والرموز.

## 🚀 التشغيل والنشر | Setup & Deployment

### التشغيل المحلي (Locally)
نظراً لاعتماد التطبيق على تقنيات الـ `Service Worker` لإتاحة العمل دون اتصال، يجب تشغيله عبر خادم ويب محلي (Web Server) بدلاً من فتح الملفات مباشرة كـ `file://`.

يمكنك تشغيله عبر إحدى الطرق التالية:

1.  **باستخدام VS Code (Live Server):**
    اضغط كليك يمين على ملف `index.html` واختر **Open with Live Server**.
2.  **باستخدام Node.js:**
    ```bash
    npm install -g http-server
    http-server .
    ```
3.  **باستخدام Python:**
    ```bash
    python -m http.server 8000
    ```

### النشر على الإنترنت (Deployment)
المشروع جاهز للنشر الفوري والمجاني على منصة **GitHub Pages** أو **Vercel** أو **Netlify** كونه يعتمد بالكامل على الملفات الثابتة (Static Files).
*   عند استخدام **GitHub Pages**: تأكد من تعيين ملف `index.html` كملف رئيسي للمستودع ليكون بمثابة صفحة الهبوط الفاخرة للزوار.

## 🧠 آلية عمل محرك الملاحة | Routing Mechanics

يعتمد منطق التطبيق البرمجي على تمثيل المحطات كـ **مخطط غير موجه (Undirected Graph)**؛ حيث تمثل كل محطة عقدة (Node)، ويمثل الاتصال بين المحطات حافة (Edge) تنتمي لخط نقل معين:

$$\text{Weight}(e) = \text{Base Station Travel Time} + \text{Line Interchange Penalty}$$

*   عند انتقال الخوارزمية من محطة إلى أخرى على نفس الخط، تحتسب التكلفة بـ `1`.
*   عند انتقال الخوارزمية بين خطين مختلفين (مثال: الانتقال من الخط الأول إلى الخط الثاني في محطة "الشهداء")، يتم تطبيق عقوبة نقل حركية بمقدار `12` وحدة افتراضية [1]. هذا يضمن ألا يقترح التطبيق مسارات تحتوي على عمليات تحويل متعددة ومزعجة إلا إذا كانت هي الخيار الوحيد رياضياً.
